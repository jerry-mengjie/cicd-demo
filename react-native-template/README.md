## 基于官方 demo，做如下优化

### RN 代码规范与格式化策略

1、启用 VSCode 保存时自动格式化，统一代码风格
2、集成 ESLint 进行代码质量检测，规范编码规范
3、使用 Prettier 自动格式化代码，保持一致的代码风格
4、引入 TypeScript（TS），提升类型安全、错误检测、开发体验
5、安装 Code Spell Checker 插件，规范英文拼写，避免拼写错误

### RN 路由优化方案

将原来注册多个 AppRegistry.registerComponent 的方式，统一调整为注册单一入口，通过参数控制显示具体页面。
优化优势：
1、路由统一由 RN 控制，减少与原生之间的耦合与通信复杂度。
2、使用 RN 自带路由栈，支持原生级流畅跳转动画，用户体验更佳。
3、事件统一处理：如返回列表时，可使用官方方法触发刷新，避免通过监听原生桥接的低效做法。
4、减少组件注册数量：有利于后续使用 CodePush 或实现 bundle 分包优化。

### Stores 模块化

1、采用 MobX 管理应用状态，简化全局状态的设计与维护，实现跨组件间的高效通信。
2、通过模块化拆分各类状态，提升代码的可读性、可扩展性与复用性。

### 主题方案优化

采用 react-navigation 官方提供的主题方案，提升主题适配的一致性与可扩展性。
1、启动页传入主题参数（如 light、dark），支持监听系统，动态切换。
2、全局使用 useTheme() 获取当前主题变量，实现样式自适应。

### 组件库建设方案

方案方向：采用「行业通用组件库 + 公司定制组件库」的组合模式，减少工作量，已获设计师初步认可。
1、RN 通用组件库： https://rn.mobile.ant.design/
2、Web 移动端通用组件库： https://mobile.ant.design/
3、公司自研组件库建设：基于通用组件进行业务抽象和样式规范，统一交互体验与品牌风格。

### Header 组件优化

基于 react-navigation 官方 Header 配置能力，统一管理导航栏样式，减少重复开发，开发人员专注页面业务。
1、通过 Stack.Navigator 设置全局 Header 样式，保持一致性。
2、各页面可使用 Stack.Screen 进行个性化配置，满足多样化场景需求。

### 多语言方案：i18next

引入主流的 i18next 解决方案，统一管理多语言资源文件，支持动态切换语言，提升国际化开发效率与可维护性。

### RN 图标管理方案：Iconfont

1、使用阿里巴巴 Iconfont 进行图标管理，集中维护项目图标资源。
2、有效减少应用包体积，提升加载性能。
